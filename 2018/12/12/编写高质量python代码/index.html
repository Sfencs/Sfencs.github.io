编写高质量python代码

发表于 2018-12-12 | 分类于 python

花了点时间将《Effective Python》读了一遍，将部分知识做了下总结

1.PEP8风格

1.函数与类两个空行隔开
2.类中方法一个空行隔开
3.赋值号左右各加一个空格
4.类与异常中单词首字母大写，如CapitalizedWord
5.模块级别常量，字母全大写加下划线ALL_CAPS
6.判断列表是否为空:if not list
7.引入模块，最好绝对路径，from a import b ，非得用相对名称就 from . import a
8.引入模块顺序:标准库，第三方，自用。每个部分安首字母顺序引入

1.当前函数作用域
2.任何外围作用域
3.包含当前代码的模块的作用域（全局作用域）

变量赋值时的规则：

3.python解释器无法严格保证private字段的私密性，其实现私有的方式其实是将私有属性的名称做了一些变换，它会在私有属性的名称前加‘_ClassName’,也就是说，如果我们使用完整的名称，那么也可以在其他地方使用私有属性。同时我们应该尽量避免使用私有字段，只有当子类不受自己控制时，才考虑使用private，避免名称冲突。
4.对于想要编写自己的容器类,可能需要编写大量的特殊方法。所以可以从collections.abc模块继承相关抽象基类，子类只需要实现基类所要求的每个方法，，其他剩余的方法由基类自动提供。如果忘记实现某个要求实现的方法，那么在使用时collections.abc模块会指出这个错误。
5.@property与@property.setter的使用

可以设置属性的set与get方法，并且可以在这基础上添加数值检查等操作
6.getattr在当前主流的Python版本中都可用，重载getattr方法对类及其实例未定义的属性有效。也就属性是说，如果访问的属性存在，就不会调用getattr方法。这个属性的存在，包括类属性和实例属性。
getattribute仅在新式类中可用，重载getattrbute方法对类实例的每个属性访问都有效。
另外，当同时定义getattribute和getattr时，getattr方法不会再被调用，除非显示调用getattr方法或引发AttributeError异常。
由于getattr只针对未定义属性的调用，所以它可以在自己的代码中自由地获取其他属性，而getattribute针对所有的属性运行，因此要十分注意避免在访问其他属性时，再次调用自身的递归循环。
当在getattribute代码块中，再次执行属性的获取操作时，会再次触发getattribute方法的调用，代码将会陷入无限递归，直到Python递归深度限制（重载setter方法也会有这个问题）。
为了避免无限递归，应该把获取属性的方法指向一个更高的超类，例如object（因为getattribute只在新式类中可用，而新式类所有的类都显式或隐式地继承自object，所以对于新式类来说，object是所有新式类的超类）。

2.multiprocessing实现的原理：

  • 1.使用pickle将数据序列化
  • 2.通过本地套接字把序列化后的数据传给子解释器在的进程
  • 3.子进程用pickle将其反序列化还原为python对象
  • 4.子进程引入相关模块来执行函数
  • 5.子进程把得到的数据序列化
  • 6.子进程通过套接字将数据传给主进程
  • 7.主进程将其反序列化为python对象
  • 8.数据合并

3.使用内置的数据结构与算法

  • 双向队列
    collections模块中的deque类

    • append(往右边添加一个元素)
    • appendleft（往左边添加一个元素
    • clear(清空队列)
    • copy(浅拷贝)
    • count(返回指定元素的出现次数)
    • extend(从队列右边扩展一个列表的元素)
    • extendleft(从队列左边扩展一个列表的元素)
    • index（查找某个元素的索引位置
    • insert（在指定位置插入元素）
    • pop（获取最右边一个元素，并在队列中删除）
    • popleft（获取最左边一个元素，并在队列中删除
    • remove（删除指定元素）
    • reverse（队列反转）
    • rotate（把右边元素放到左边）

  • 有序字典
    collections模块中的OrderedDict类

  • 带有默认值的字典
    collections模块中的defaultdict类

    该类对象的创建为dict =defaultdict( factory_function)，这里的factory_function可以为list、set、str、int等，当访问字典中的某个键但其没有值时，返回的是factory_function的默认值，如int对应0，list对应[],str对应’’等

  • 堆队列，（优先级队列）

    heapq模块提供了heappush、heappop、nsmallest等函数，能够在标准的list类型之中创建堆结构

  • 二分查找

    bisect模块中的bisect_left函数能够在一系列排好顺序的元素中搜寻某个值,这种算法的复杂度是对数级别的

  • itertools模块中含有大量可以用来组合并操控迭代器的函数，要做一些非常复杂的迭代过程的程序时，先来看看itertools中是否有现成的
